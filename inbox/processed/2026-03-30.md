## Qué aprendí hoy

- Puedo convertir string a entero dígito a dígito: num = num * 10 + (c - '0'), sin necesidad de stoi/stoll
- En Basic Calculator lo que importa guardar en el stack es el signo acumulado, no el total ni el signo anterior
- Como las operaciones son solo suma y resta, el orden no importa — puedo sumar directo al total en cualquier momento
- El manejo de signos con enteros (1 y -1) y multiplicaciones es más limpio que manejar chars
- Los signos negativos unarios no necesitan manejo especial — el stack de signos ya los resuelve solo
- No necesito limpiar el string antes del loop — puedo hacer continue si encuentro espacio

## Errores que cometí

- Primera solución: parchada, modificaba el string antes de procesarlo para manejar negativos unarios
- Segunda solución: usaba stoi que tronaba con overflow en números grandes, parchado con stoll
- No vi desde el inicio que el signo acumulado era lo que necesitaba guardar en el stack

## Código

// Solución canónica final
class Solution {
public:
    int calculate(string s) {
        int sign = 1;
        int total = 0;
        stack<int> backSign;
        backSign.push(1);
        unsigned int num = 0;
        for (int i = 0; i < s.size(); i++) {
            if(s[i] == ' ') continue;
            if (isdigit(s[i])) {
                num = num *10 + (s[i] - '0');
            } else if (s[i] == '(') {
                backSign.push(backSign.top()* sign);
                sign = 1;
            } else if (s[i] == ')') {
                total += num *sign* backSign.top();
                backSign.pop();
                sign = 1;
                num = 0;
            } else {
                total += num *sign* backSign.top();
                sign = (s[i] == '+') ? 1 : -1;
                num = 0;
            }
        }
        total += num *sign* backSign.top();
        return total;
    }
};
