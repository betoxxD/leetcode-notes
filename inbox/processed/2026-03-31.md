# 2026-03-31 — Linked List Cycle

## Problemas trabajados

- Linked List Cycle — Easy — <https://leetcode.com/problems/linked-list-cycle>

## Tiempo invertido

1 hora en total, 15 min en solución simple (con hash set) y 45 min en investigar y entender el proceso de liebre/tortuga para O(1) de espacio + refrescar conceptos de linked list.

## Qué aprendí hoy

Aprendí a aplicar el algoritmo de liebre/tortuga, además de entender por qué es que este funciona. Esto no es aprendizaje nuevo, pero si es refrescar conocimientos de mucho tiempo atrás, la parte de las Linked List, cómo moverme por cada nodo y cómo se utilizan los punteros para utilzar las listas enlazadas. Además, entendí la diferencia entre un puntero y una referencia, el puntero es un apuntado a una dirección de memoria y la referencia es eso, se parecen, pero la diferencia es que la referencia no se puede reasignar ni reutilizar.

## Errores que cometí

El único error que cometí fue no validar el head, me dió TLE por eso.

## Código (opcional)

```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode(int x) : val(x), next(NULL) {}
 * };
 */
class Solution {
public:
    bool hasCycle(ListNode* head) {
        ListNode *slow, *fast;

        if (head == nullptr) {
            return false;
        }

        slow = head;
        fast = head;

        while (fast->next != nullptr && fast->next->next != nullptr) {
            fast = fast->next->next;
            if (slow == fast) {
                return true;
            }
            slow = slow->next;
        }

        return false;
    }
};
```

# 2026-03-31 — Merge Two Sorted Lists

## Problemas trabajados

- Merge Two Sorted Lists — Easy — <https://leetcode.com/problems/merge-two-sorted-lists/description/>

## Tiempo invertido

30 minutos en total, 15 min en solución rápida y 15 min en entender y armar la solución canónica.

## Qué aprendí hoy

Puedo evitar el p1 != nullptr utilzando solamente if(p1). Cuando utilizo una linked list, puedo reutilizar los nodos para terminar la lista, por ejemplo, en este problema cuando ya no existe uno de los 2 nodos, puedo simplemente hacer dummy->next = nodoExistente.

## Errores que cometí

Tratar las linked lists como arreglos. Hacer el ciclo completamente cuando pude haber terminado cuando alguno de las 2 listas termina y solamente enlazo la siguiente.

## Código (opcional)

```cpp
class Solution {
public:
    ListNode* mergeTwoLists(ListNode* list1, ListNode* list2) {
        ListNode sorted;
        ListNode* dummy = &sorted;

        while (list1 && list2) {
            if (list2->val < list1->val) {
                dummy->next = list2;
                dummy = list2;
                list2 = list2->next;
            } else {
                dummy->next = list1;
                dummy = list1;
                list1 = list1->next;
            }
        }

        dummy->next = list1 ? list1 : list2;

        return sorted.next;
    }
};
```

# 2026-03-31 — Add Two Numbers

## Problemas trabajados

- Add Two Numbers — Medium — <https://leetcode.com/problems/add-two-numbers/description/>

## Tiempo invertido

40 minutos en total, 20 min en hacer la solución sucia y 20 min en hacer la solución canónica.

## Qué aprendí hoy

Primero que nada, el tener Object *a; y Object* b; y al hacer a = b, lo que pasa es que a apunta a la dirección de memoria en b. Si se crea Object *c y ahora hago a = c, no se estaría cambiando el contenido de a, sino que se estaria cambiando el puntero a. Esto dió como resultado llegar a la solución de dummy head, la cuál es crear un Nodo sentinela y un nodo dummy, el dummy se iguala al sentinela y se va recorriendo por el listado utilizando y modificando el dummy.

## Errores que cometí

Porque estaba sobrecomplicando la lógica, tenía un while(true) y controlaba el flujo mediante un if dentro del while. Además, estaba haciendo una asignación sobrecomplicada, la cuál iba de tener un current que se inicializa con result, luego a ese current dentro del flujo se le agregaba el total y luego ahí mismo se creaba el nuevo nodo y se asignaba a current next, esto me daba al final un nodo que no debería de ir, entonces es ahí donde entra el romper el ciclo antes de hacer eso, y funcionó pero no es lo más límpio en código. Además, el sobrante se agregaba igualmente dentro del ciclo al hacer el break.

## Código (opcional)

```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    ListNode* addTwoNumbers(ListNode* l1, ListNode* l2) {
        ListNode* result = new ListNode();
        ListNode *newNode, *backNode, *dummy;
        int n1, n2, total, carry = 0;

        dummy = result;

        while (l1 != nullptr || l2 != nullptr) {
            n1 = l1 != nullptr ? l1->val : 0;
            n2 = l2 != nullptr ? l2->val : 0;
            total = n1 + n2 + carry;

            if (total >= 10) {
                total %= 10;
                carry = 1;
            } else {
                carry = 0;
            }

            newNode = new ListNode(total);

            dummy->next = newNode;
            dummy = newNode;

            if (l1 != nullptr) {
                l1 = l1->next;
            }

            if (l2 != nullptr) {
                l2 = l2->next;
            }
        }

        if (carry) {
            newNode = new ListNode(carry);
            dummy->next = newNode;
        }

        return result->next;
    }
};
```
